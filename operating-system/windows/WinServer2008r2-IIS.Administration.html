# Windows Server 2008 R2 安装 IIS.Administration

  • 当被告知IISAdministrationSetup安装失败，我的内心毫无波澜，甚至想问你有没有试过管理员权限安装，因为在其它相同系统服务器成功安装过，所以觉得问题不大，但是仍需解决啊，下面先给出解决办法，再进行分解

# 先给出解决办法

  • 安装补丁，重启电脑
  • 我的是win server 2008 r2，所以选择这个补丁
  • 管理员身份运行安装，记住不要以兼容模式运行，否则即使显示安装成功也不是真的安装成功
  • 因为这个补丁可以让dotnet命令成功执行，所以得以成功安装。下面记录一下过程。

# 分析

  • 安装窗口提示Error 0x80070643，点击日志文件链接，查看日志，发现是错误是iis_administration_msi这个msi包安装失败，但是具体原因没说
  • 百度谷歌了一堆Error 0x80070643: Failed to install MSI package都没有结果对不对，我也是。中间有个博客是说它出错的软件要将域管理员切换本地管理员，操作是：按住Shift键=>右键单击安装可执行文件=>以其他用户身份运行=>然后输入本地管理员（PC名称\管理员）。试过了也不行，说明不是权限的问题。
  • 去官网文档、github主页除了确认它支持Windows Server 2008 R2以及更高版本，就没其他信息了。 仅仅这些信息，实在不能分析出什么，只能靠猜了

# 尝试

  • IISAdministration是基于asp.net core来做的，查看程序和功能，可以看到它已经成功安装.net core runtime了，这个环境依赖应该没问题。但我还是安装了一个对应版本的.net core sdk，我下载了.net core runtime 1.07对应的版本
  • 结果当然还是不行啦，猜想iis_administration_msi安装失败会不会是兼容性问题，于是我就用兼容模式安装，选中安装包-右键点击属性-兼容性选项卡-勾选以兼容模式运行-选择winserver2008，再加上管理员身份运行，居然！显示安装成功了
  • 然而并没有什么用，C:\Program Files\IIS Administration并没有文件，因此站点一直访问不了https://localhost:55539/
  • 经过多次卸载再安装，还因为它安装卡住了重启了一次之后，还是无果。既然没有文件，那我给你一份呗我自己启动运行呗，反正是asp.net core项目。于是从本地电脑拷贝一份过去，在C:\Program Files\IIS Administration\2.2.0\Microsoft.IIS.Administration目录按住shift+鼠标右键-打开命令窗口-执行命令dotnet Microsoft.IIS.Administration.dll，结果报Failed to load the dll from [C:\Program Files\dotnet\host\fxr\1.1.2\hostfxr.dll]这个错
  • 发现只要输入dotnet命令，都是这个错，sdk安装成功了，但是命令不执行，查了一下发现需要安装补丁，参见此博客，附补丁地址
  • 装完补丁重启电脑，重新输入命令，竟然跑起来了，但是https://localhost:55539/依旧访问不了，想想这种方式即使得了也不靠谱，毕竟还有一个msi包还没安装，设置都没设置好，再次尝试安装，还是失败

# 正确的姿势

  • 实在不行，只能继续分析日志了，但是那个面板的日志文件链接打开能看到的错误信息有限啊，于是我想到查看成功安装的日志是什么样的呢，可以对比一下，于是找一下他的路径
  • 在打开日志文件的时候，右键单击记事本，出现一排文件记录，找到这个日志文件，鼠标放在上面即可看到文件路径，我的位于C:\Users\Administrator\AppData\Local\Temp，可以看到，除了之前看到的日志文件，还有一个后面多了_000_iis_administration_msi的文件，心里一喜，立马打开，拉到最下面，一般出错了就会中止安装，错误日志记录在最下面
  • 只能看到Product: Microsoft IIS Administration -- Installation failed以及安装成功或错误状态: 1603，在往上翻，突然注意到了C:\Program Files\IIS Administration这个路径，这个不是被我拷贝了一份在那里吗，会不会影响了。立马删了再次安装，哎呦我去！成功了，并且正常访问https://localhost:55539/，服务里面也多了Management service for IIS这一项，这次才是成功！

# 总结

  • 这次可以说是误打误撞解决了问题，浪费了很多时间，但也学到不少东西，起码不会干瞪眼着急了

  • 首先安装大型一点的软件，如果出错了，一般都会有日志文件的，分析一下，找到问题所在，针对性解决就可以了，如果找不到问题或者不理解那个问题，说明我们对这个东西所涵括的内容，运行流程还不太了解，大概了解一下有时候能帮助我们及早解决问题。

  • 一般安装出错，error会出现在日志文件最后面，仔细检查每一个error，分析出问题所在，如果信息太少，可以查看日志文件所在位置，是否还有其他日志文件。

  • 一般主日志文件会有记录所有日志文件的地址，细心看一下就可以找到是否还有其它日志文件，找到之后继续分析关键点。找到关键的名词之后，查一下相关的资料，发散思维去解决问题，你会有意想不到的收获。

  • 如果是安装或者部署软件出现问题，最好可以调试一下，一步步调试，直面运行过程，可以发现很多我们没想过的细节，经过一番努力，对这些知识的印象也会更加深刻。

  • 不放弃，最后才能成功并有收获！
